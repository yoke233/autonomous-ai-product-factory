# Autonomous AI Product Factory

系统接受一份委托，围绕它安排工作、收集产出、作出判定；获授权的外部作用产生新的产出，直到系统能够确认结果。本文档只定义项目通用语言和不变量，不规定数据库、状态机或具体实现。

## 核心语言

**委托（Mandate）**：
人发布给系统的一份不可变、可版本化约定，固定期望、事实基础、权限、预算、判断标准和交付目标。任何影响结果适用性的已发布内容变化都产生新修订。
_Avoid_: 可变需求单、浮动的“当前要求”

**工作（Work）**：
跨多次尝试持续存在、具有稳定身份和明确解除条件的可调度义务。工作完成只表示义务已经产生规定产出或终止信息，不表示结果已被接受或已经交付。
_Avoid_: 一次运行等同工作、任务状态等同产品状态

**产出（Result）**：
工作或外部观察产生的不可变、带来源内容。产出只提供信息，不授予权限，也没有 accepted、rejected、stale 或 delivered 状态。
_Avoid_: Agent 声明、测试输出或外部回执直接成为权威事实

**判定（Judgment）**：
有权判定者在精确委托修订下，针对一项产出并参考其他产出发布的不可变权威结论。改判、补证和撤销产生新判定，不能覆盖原判定或改写委托与产出。
_Avoid_: 最后写入状态、工具输出、LLM 自评或多数投票直接成为裁决

**作用（Effect）**：
由判定固定身份、完整规范并授权的单个外部动作意图。执行尝试、回执、现实确认和补偿均与原作用分开记录。
_Avoid_: 把授权、调用、回执和现实结果混成一个状态

## 运行语言

**尝试（Attempt）**：
执行某项工作或作用的一次持久化运行，承担重试来源、超时、执行 lineage、lease 和 fencing。尝试成功只表示本次执行结束。

**材料引用（Artifact Ref）**：
产出指向 patch、代码、制品、日志或报告等不可变材料的稳定引用与内容摘要。材料本身没有裁决权。

**执行适配器（Execution Adapter）**：
接受工作或作用并产生尝试与产出的可替换实现，包括 Agent、Worker、编排器和外部平台。执行适配器不能发布判定。

## 权威来源

- 人发布和修订委托，并保留接受最终结果、授权外部作用和确认交付的权力。
- 委托可以把范围明确、结果确定的规则委托给系统，例如在完成条件满足时解除工作，或在强制检查失败时自动拒绝。
- LLM、Agent、Worker、测试器和外部平台只能产生或转交产出；它们的输出不能自行变成判定。
- 控制面验证判定者权限、固定引用并保存判定，但控制面这个软件模块本身不是权威来源。

## 核心关系

```text
Mandate 约束并派生 Work
Work 经 Attempt 产生 Result
Judgment 在精确 Mandate 下裁决 Result
Judgment 必要时固定并授权 Effect
Effect 经 Attempt 产生新的 Result
Judgment 对外部现实和最终结果作确认
```

## 不变量

- 委托修订、产出、判定和作用一经发布均不可原位修改。
- 工作跨尝试持续存在；尝试只记录一次执行。
- 所有引用固定到精确身份、修订或内容摘要，不使用浮动对象。
- 工作完成、产出被接受和目标已交付是不同判定，不能由一个 `SUCCESS` 混合表达。
- 一个作用只表达一个规范化外部动作；重试沿用同一作用身份和幂等域。
- 作用超时表示结果未知；必须回读、对账或保留未知，不能直接创建新作用重发。
- 补偿是引用原作用、经过新授权的另一个作用。
- 历史上发生过、历史上曾被接受和当前仍然适用是三个不同命题。
- CI/CD、Git 和部署平台可以执行作用或提供产出，但不进入领域内核。
